# Full command test coverage

## Goal

Integration tests for all CLI commands, generated using the QA test creator skill.

## Decisions & constraints

- Covers: `source init/list/remove`, `install`, `uninstall`, `update`, `merge` (with conflict, `--continue`, `--abort`, `--rebase`, orphan, mergeable), `status`, `provider add/list/remove`.
- Tests use primitives for edge cases, presets for common setups.
- Generated via `/qa-test-creator` skill — invoke the skill with scenario descriptions for each command group.

## Edge cases

- Detached skills (commit unreachable from pinned branch)
- Broken sources (missing config)
- Multiple providers with diverged edits
- Orphan skills, mergeable skills
- Merge conflict resolution flow (`--continue`/`--abort`)
- Multiple sources requiring `--source` disambiguation
- `update --offline` skip behavior

## Key files

- `integration/tests/test_source.py` (new)
- `integration/tests/test_provider.py` (new)
- `integration/tests/test_update.py` (new)
- `integration/tests/test_merge.py` (new)
- `integration/tests/test_status.py` (new)
- `integration/tests/test_install.py` (new)

## Acceptance criteria

- Every CLI command has at least one happy-path integration test
- Edge cases listed above are covered
- All tests run via `./integration/run.sh test` and produce inspectable logs
- Tests generated by QA test creator skill follow framework conventions
