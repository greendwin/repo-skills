# Wrapper-based tsc check wired in, raw file passing at baseline

## Goal

The `js` tox env runs `tsc -p .claude/workflows` (before `node --test`) and is green. This proves the wrapper+tsc pipeline end-to-end against the still-unannotated `grind-story.js`: only the structural blockers (top-level return/await) and the injected globals are resolved, so any red in later slices comes from typing, not plumbing. No `// @ts-check` and no domain typedefs yet — `checkJs` will report nothing for the raw helpers because they carry no annotations, but the wrapper must parse and the globals must resolve.

## Decisions & constraints

- **Make the function-body file checkable via a generated wrapper.** `tsc` rejects the raw file — empirically: TS1375 (top-level `await` needs a module), TS1108 (top-level `return` illegal), TS2304 (undeclared globals). Its suggested fix (`export {}`) would break the runtime, and there is no global per-error-code suppression. So add `.claude/workflows/check-types.mjs` that reads `grind-story.js`, wraps its content as `export async function __wf(){ <body> }` into a gitignored `*.wrapped.mjs`, then invokes `tsc -p`. Put the wrapper prefix on line 1 (joined with the original first line) so reported line/column numbers stay aligned with the source. The shipped `grind-story.js` stays byte-for-byte a bare function body. *Rejected: restructuring into `async function main()` + `return await main()` (final line is still top-level return+await); checking only helpers extracted to a sidecar (runtime loads one self-contained body, cannot import).*
- **Committed `globals.d.ts` declares the injected runtime API.** Ambient declarations for `agent`, `parallel`, `pipeline`, `phase`, `log`, `args`, `budget`, `workflow` — resolving TS2304 and documenting the Workflow API surface. Give them real signatures (e.g. `agent(prompt: string, opts?: {...}): Promise<any>`, `parallel`, `pipeline`, `log(msg: string): void`, `phase(title: string): void`, `args: any`, `budget: {...}`). Scoped to the runtime API only.
- **Committed `tsconfig.json`, `strict: true`, excluding the raw file.** `allowJs`, `checkJs`, `noEmit`, es2022 target/module (top-level await in the wrapper), `types: ["node"]`. It must EXCLUDE the raw `grind-story.js` (would re-hit TS1108) and INCLUDE `globals.d.ts`, the generated `*.wrapped.mjs`, all `*.test.mjs`, `_extract-fn.mjs`, and `check-types.mjs`. Mirrors the repo's `mypy strict`. *Rejected: pure CLI flags (long, awkward exclusion, no in-IDE checking).*
- **`tsc` checks the whole `.claude/workflows/` JS surface**, requiring the pinned `@types/node` (added in Slice 1). The `.test.mjs`/`_extract-fn.mjs`/`check-types.mjs` files are plain ES modules (no wrapper needed) but use node builtins. They must type-clean at baseline too — if any needs a small annotation to pass strict, that is in scope here.

## Edge cases

- The generated `*.wrapped.mjs` must be regenerated by `check-types.mjs` each run (do not commit it; `.gitignore` already covers it).
- `tsconfig` glob must pick up the generated wrapper inside `.claude/workflows/` while excluding the raw `grind-story.js` — verify the include/exclude interplay actually excludes the raw file (TS1108 must not appear).
- `strict: true` over the test files may surface implicit-any in `_extract-fn.mjs`'s `new Function` slicing — annotate minimally to pass, do not weaken strict.
- Order in the `js` env: preflight → `npm ci` → `tsc` (fail fast) → `node --test`.

## Key files

- `.claude/workflows/check-types.mjs` — new: generate wrapper, run `tsc -p`.
- `.claude/workflows/globals.d.ts` — new: ambient runtime API.
- `.claude/workflows/tsconfig.json` — new: strict, checkJs, excludes raw file.
- `tox.ini` — add the `tsc` invocation to the `js` env (before `node --test`).
- `.claude/workflows/*.test.mjs`, `_extract-fn.mjs` — annotate only if strict requires it to pass.

## Acceptance criteria

- `uv run tox -e js` runs `tsc -p .claude/workflows` and it is green; `node --test` still runs after and is green.
- Pointing `tsc` at the raw `grind-story.js` directly still produces TS1108 (proving the exclusion + wrapper are doing the work), but the env's wrapped check is clean.
- `globals.d.ts` resolves all eight injected globals (no TS2304 in the wrapped output).
- The generated `*.wrapped.mjs` is not committed; the line-1-joined wrapper keeps error line numbers aligned (spot-check by introducing a temporary type error and confirming the reported line matches the source).
